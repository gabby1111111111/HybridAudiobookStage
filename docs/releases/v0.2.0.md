# v0.2.0 - 轻量 TTS 首个正式版本

发布日期：2026-07-21

`v0.2.0` 是 HybridAudiobookStage 以“轻量 TTS”为核心的首个正式版本。它把旁白、角色台词、单句、全文、选中文字和当前段落统一到同一套路由、播放队列与持久缓存中，同时保留原有视频字幕舞台。

## 新增

- 三步轻量配置流程，日常使用不需要理解高级 Provider 参数。
- Edge、IndexTTS2 / OpenAI 兼容、豆包原生三类 TTS Provider。
- 旁白、角色默认、全文单音色和特定角色覆盖路由。
- 全文、单句、选中文字、当前段落和声音试听入口。
- Edge 302 个音色目录、豆包猫箱 45 个音色目录和 IndexTTS2 `.wav` 目录发现。
- 可搜索音色选择器，支持名称、ID、Locale、Gender、男声和女声关键字。
- 统一缓存：内存 → IndexedDB → SillyTavern 服务器共享缓存 → Provider。
- PC 和手机共享生成语音，以及缓存自检、统计和清理。
- 台词按钮缓存状态、暂停/继续、统一 Playback Session 和后续分段预取。
- 完整中文 README、安装说明、使用指南和常见问题。

## 主要改进

- 单句、全文和精确选区对同一句话复用同一个缓存，不再因为入口不同重复生成。
- 页面刷新后优先命中 IndexedDB；其他设备可以命中服务器共享缓存。
- 音色下拉框显示可读名称与完整元数据，长 ID 可以换行，不再只露出尾部残片。
- 每个 Profile 记住上次选择的音色，Provider 之间不会串音色。
- Markdown 强调符号不会进入合成文本，例如 `*但真的好听吗？*` 只朗读正文。
- 新朗读、Stop、切换聊天、编辑或 Swipe 会取消旧请求，旧声音不会稍后复活。

## 豆包说明

- 使用固定的火山引擎 v3 单向 TTS 接口。
- APP ID、Access Key 和 Resource ID 由 SillyTavern 服务器插件转发。
- 支持 `seed-tts-2.0` 和 `seed-icl-2.0`。
- 猫箱同款 `ICL_uranus_*` 音色需要 `seed-icl-2.0`。
- Access Key 不进入缓存身份、Audit 或用户可见错误。

## 安装提示

前端扩展可以通过 SillyTavern 的“安装扩展”功能安装本仓库。豆包原生、手机访问本机 IndexTTS2、服务器共享缓存和 Index 音色目录发现还需要安装仓库中的服务器插件：

```text
server-plugin/HybridAudiobookStage-Launcher
```

服务器插件安装或更新后必须重启 SillyTavern。

## 升级与兼容

- 旧版 IndexTTS2 和 Edge 配置会迁移为保留的 Provider Profile。
- 特定角色旧 Index 覆盖可能继续高于角色默认路由；轻量设置页会显示警告，并提供需要确认的清理按钮。
- 不会自动删除豆包、Edge 或其他用户明确配置的特定角色覆盖。
- 不会清除已有音频缓存和用户资产。

## 已知限制

- 豆包第一阶段使用服务默认合成语速；快慢可以先通过播放器倍速调整。
- Edge 需要兼容的 SillyTavern Edge TTS 服务器插件。
- IndexTTS2 真正试听和朗读前仍需要启动本机 API。
- API Key 与 Access Key 保存在 SillyTavern 共享扩展设置中，界面遮罩不等于加密。
- 当前仓库尚未附带开源许可证。

完整用法请阅读仓库首页的中文 README 和 `docs/轻量TTS使用指南.md`。
